# Documentation Skill

## Purpose

Keep CompText documentation concise, navigable, and aligned with current implementation status.

## When to use

Use for README, architecture docs, operational instructions, and documentation-only changes.

## Required checks

- Keep `AGENTS.md` minimal and operational.
- Put architecture detail in `docs/`.
- Run markdown or diff checks when available.

## Safety notes

- Do not claim planned contracts are implemented.
- Do not claim production, compliance, security, forensic, provider, or release readiness.
- Do not include secrets, raw environment variables, or provider payloads.
